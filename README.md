# scr-med-inf.sh

Script for acquiring information from a storage device as part of digital investigations.

## Details

The desired effect is to immediately obtain the technical information of a physical media device, as well as the partitions whose presence is detected. The mechanisms of this script are based on common Linux commands (details in the "Details" section) requiring minimal user intervention.

## Details

- Most of the script's commands are able to maintain a consistent display despite the absence of results;
- The "*" character prefixing certain results specifies that they are obtained by calculation and not by direct reading;
- For logging purposes and subsequent analysis, the script records the results of commands relating to all storage devices, in dedicated text files;
- The commands used are as follows: **fdisk**, **hdparm**, **lsblk**, **lshw**, **smartctl**, **testdisk** and **udevadm**. The availability of these commands within the system is validated at the start of script execution, and the versions identified are specified;
- The results of interest are formatted on screen, and saved simultaneously in a file, using the **tee** command when the script is called (see "Operation" for details).

## Operating perimeter

The script is validated on the following types of device:
- [x] MMC cards ;
- [x] SD cards ;
- [x] USB keys;
- [x] Hard disks.

## Operation

Activate script execution :
```
chmod +x scr-med-inf.sh
```

Execute script:
```
./scr-med-inf.sh
```

Script execution, simultaneous saving in a file:
```
./scr-med-inf.sh | tee <FICHIER>
```
